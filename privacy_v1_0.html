Используя Приложение, Вы даете свое согласие на обработку персональных данных в порядке и на условиях, определенных настоящей Политикой.

3. ОПРЕДЕЛЕНИЯ

В целях настоящей Политики используются следующие определения:

  • Персональные данные – любая информация, относящаяся к прямо или косвенно определенному или определяемому физическому лицу (субъекту персональных данных);
  • Оператор – лицо, самостоятельно или совместно с другими лицами организующее обработку персональных данных, а также определяющее цели обработки персональных данных;
  • Обработка персональных данных – любое действие или совокупность действий, совершаемых с персональными данными, включая сбор, запись, систематизацию, накопление, хранение, уточнение, извлечение, использование, передачу, обезличивание, блокирование, удаление, уничтожение;
  • Пользователь – лицо, использующее Приложение.

4. КАТЕГОРИИ ОБРАБАТЫВАЕМЫХ ПЕРСОНАЛЬНЫХ ДАННЫХ

При использовании Приложения обрабатываются следующие категории персональных данных:

4.1. Данные об автомобилях:

  • Марка и модель автомобиля;
  • Год выпуска;
  • Государственный регистрационный номер транспортного средства (опционально);
  • Объем двигателя;
  • Тип топлива;
  • Тип коробки передач;
  • Цвет автомобиля;
  • Начальный пробег;
  • Прозвище автомобиля (опционально).

4.2. Данные о расходах:

  • Категория расхода;
  • Сумма расхода;
  • Дата расхода;
  • Описание расхода;
  • Пробег на момент расхода (опционально);
  • Фотографии чеков и документов (опционально).

4.3. Технические данные:

  • Настройки приложения;
  • Данные о подписке (тип подписки, дата окончания, статус);
  • Токены доступа к сервису Яндекс.Диск (опционально, для функции резервного копирования);
  • Данные о версии приложения;
  • Настройки виджетов;
  • Настройки уведомлений.
Важно: Приложение не собирает и не обрабатывает данные, позволяющие идентифицировать личность пользователя (ФИО, номер телефона, адрес электронной почты, паспортные данные и т.п.), за исключением случаев, когда пользователь самостоятельно указывает такие данные в описании расходов или других полях Приложения.

5. ЦЕЛИ ОБРАБОТКИ ПЕРСОНАЛЬНЫХ ДАННЫХ

Обработка персональных данных осуществляется в следующих целях:

  1. Предоставление функционала Приложения для учета расходов на автомобиль;
  2. Формирование статистики и отчетов о расходах;
  3. Обеспечение работы функции резервного копирования данных на Яндекс.Диск (опционально);
  4. Улучшение работы Приложения и разработка новых функций;
  5. Обеспечение безопасности использования Приложения.

6. ПРАВОВЫЕ ОСНОВАНИЯ ОБРАБОТКИ ПЕРСОНАЛЬНЫХ ДАННЫХ

Обработка персональных данных осуществляется на основании согласия субъекта персональных данных на обработку его персональных данных (пункт 1 части 1 статьи 6 Федерального закона № 152-ФЗ).

Согласие на обработку персональных данных предоставляется пользователем путем установки и использования Приложения. Использование Приложения означает принятие условий настоящей Политики.

7. СПОСОБЫ И СРОКИ ОБРАБОТКИ ПЕРСОНАЛЬНЫХ ДАННЫХ

7.1. Способы обработки:

  • Сбор: Персональные данные собираются непосредственно в Приложении при их вводе пользователем;
  • Хранение: Персональные данные хранятся локально на устройстве пользователя в базе данных SQLite (Room Database) и в файлах настроек (SharedPreferences);
  • Использование: Персональные данные используются для предоставления функционала Приложения, формирования отчетов и статистики;
  • Передача: Персональные данные могут передаваться на серверы Яндекс.Диск только при использовании функции резервного копирования и только с явного согласия пользователя;
  • Уничтожение: Персональные данные удаляются при удалении соответствующих записей в Приложении или при удалении Приложения с устройства пользователя.

7.2. Сроки обработки:

Персональные данные обрабатываются в течение срока действия согласия на их обработку, а именно:

  • В течение всего периода использования Приложения пользователем;
  • До момента отзыва согласия на обработку персональных данных (путем удаления Приложения с устройства);
  • После удаления Приложения все данные удаляются с устройства пользователя автоматически.

В случае использования функции резервного копирования на Яндекс.Диск, данные хранятся на серверах Яндекс.Диск в соответствии с политикой конфиденциальности Яндекс (https://yandex.ru/legal/confidential/).

8. ПЕРЕДАЧА ПЕРСОНАЛЬНЫХ ДАННЫХ ТРЕТЬИМ ЛИЦАМ

Оператор не передает персональные данные третьим лицам, за исключением следующих случаев:

  1. При использовании функции резервного копирования данных на Яндекс.Диск (опционально) персональные данные передаются на серверы ООО «Яндекс» в зашифрованном виде в соответствии с политикой конфиденциальности Яндекс;
  2. По требованию уполномоченных органов государственной власти в случаях, предусмотренных действующим законодательством Российской Федерации.
Важно: Оператор не передает персональные данные третьим лицам в коммерческих целях и не использует их для рекламы без согласия пользователя.

9. МЕРЫ ПО ОБЕСПЕЧЕНИЮ БЕЗОПАСНОСТИ ПЕРСОНАЛЬНЫХ ДАННЫХ

Оператор применяет следующие меры для защиты персональных данных:

  • Хранение персональных данных осуществляется исключительно на устройстве пользователя в зашифрованной базе данных;
  • При передаче данных на серверы Яндекс.Диск используется защищенное соединение (HTTPS) и токены доступа OAuth;
  • Приложение не требует регистрации и не собирает идентифицирующие пользователя данные;
  • Доступ к данным приложения на устройстве защищается средствами операционной системы Android;
  • Все разрешения запрашиваются у пользователя явно и используются только для необходимых функций.

11. РАЗРЕШЕНИЯ ПРИЛОЖЕНИЯ

Приложение запрашивает следующие разрешения операционной системы Android:

  • INTERNET – для обеспечения работы функции резервного копирования на Яндекс.Диск (используется только при включении данной функции);
  • ACCESS_NETWORK_STATE – для проверки доступности интернет-соединения;
  • CAMERA – для съемки фотографий чеков и документов (запрашивается только при использовании функции добавления фото);
  • READ_EXTERNAL_STORAGE / READ_MEDIA_IMAGES – для доступа к фотографиям на устройстве (для Android версий до 13 и начиная с Android 13 соответственно, запрашивается только при необходимости);
  • WRITE_EXTERNAL_STORAGE – для сохранения файлов экспорта (для старых версий Android, запрашивается только при экспорте данных).
Важно: Все разрешения запрашиваются у пользователя явно в момент необходимости их использования. Пользователь может отказать в предоставлении разрешений, что может ограничить функциональность некоторых функций Приложения.

12. РЕЗЕРВНОЕ КОПИРОВАНИЕ ДАННЫХ

Приложение поддерживает опциональную функцию резервного копирования данных на сервис Яндекс.Диск. Данная функция:

  • Включается пользователем вручную в настройках Приложения;
  • Требует авторизации через OAuth 2.0 на сервисе Яндекс.Диск;
  • Передает данные на серверы ООО «Яндекс» в зашифрованном виде;
  • Осуществляется в соответствии с политикой конфиденциальности Яндекс (https://yandex.ru/legal/confidential/);
  • Может быть отключена пользователем в любой момент, что приведет к удалению данных с серверов Яндекс.Диск (если пользователь не сохранил их вручную).
